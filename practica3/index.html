ESP32 Thingspeak IoT Sensores

- Resumen -

En esta práctica se explora la integración de la plataforma ThingSpeak con módulos ESP32/ESP8266 para la monitorización de sensores en tiempo real. Se implementan cuatro tipos de sensores: un botón (sensor digital), un sensor PIR (sensor digital de movimiento), un potenciómetro (sensor analógico) y un sensor DHT22 (sensor inteligente de temperatura y humedad). Los datos recopilados por estos sensores son transmitidos a través de WiFi a la plataforma ThingSpeak, donde se visualizan en gráficos en tiempo real, demostrando así las capacidades de IoT de estos módulos y la versatilidad de la plataforma de monitorización en la nube.

- Introducción -

El Internet de las Cosas (IoT) ha transformado la manera en que interactuamos con nuestro entorno, permitiendo la recolección, transmisión y análisis de datos en tiempo real. En este contexto, los módulos ESP32 y ESP8266 se han convertido en componentes fundamentales gracias a su capacidad de conectividad WiFi, bajo costo y versatilidad.

ThingSpeak es una plataforma de análisis de IoT que permite agregar, visualizar y analizar flujos de datos en vivo en la nube. Esta plataforma ofrece visualizaciones instantáneas de los datos enviados por los dispositivos y permite la creación de aplicaciones IoT sin necesidad de configurar servidores.

En esta práctica, integraremos cuatro tipos diferentes de sensores con el ESP32 o ESP8266 y enviaremos sus lecturas a ThingSpeak para su visualización. Cada sensor representa un caso de uso distinto: el botón como entrada digital simple, el sensor PIR para detección de movimiento, el potenciómetro como entrada analógica variable, y el sensor DHT22 para mediciones ambientales precisas. Esta combinación nos permitirá comprender el flujo completo desde la adquisición de datos físicos hasta su visualización en una plataforma IoT.

- Materiales -

  • Módulo NodeMCU ESP8266 o ESP32
  • Botón pulsador (para sensor digital)
  • Resistencia de 10K ohms (pull-down para el botón)
  • Sensor PIR (HC-SR501)
  • Potenciómetro de 10K ohms
  • Sensor DHT22 (temperatura y humedad)
  • LED y resistencia de 220 ohms (indicador visual)
  • Protoboard y cables de conexión
  • Cuenta en la plataforma ThingSpeak
  • Entorno Arduino IDE configurado con bibliotecas ESP32/ESP8266 y DHT

- Desarrollo -

1. Configuración de ThingSpeak

  1. Crear una cuenta en ThingSpeak (thingspeak.com)
  2. Generar un nuevo canal con cuatro campos:
    • Field 1: Sensor Digital (Botón)
    • Field 2: Sensor Digital PIR
    • Field 3: Sensor Analógico (Potenciómetro)
    • Field 4: Sensor Inteligente (Temperatura DHT22)
  3. Obtener la API Key para escritura en el canal

2. Configuración del IDE Arduino

  1. Instalar las bibliotecas necesarias:
    • ESP8266WiFi/WiFi (dependiendo del módulo usado)
    • DHT
    • ThingSpeak

3. Sensor Digital (Botón)

Simulación en WOKWI:

Código Sensor Digital (Botón) 
#include <WiFi.h>
#include <HTTPClient.h>

int buttonPin = 26;
int ledPin = 27;
int buttonState = 0;

String url = "https://api.thingspeak.com/update?";
String api_key = "FUV135ZIDNU0UE1K";
String sensor = "field1";

void setup() {
  pinMode(buttonPin, INPUT);
  pinMode(ledPin, OUTPUT);
  Serial.begin(115200);
  
  WiFi.begin("ALFREDOMACHINE", "35|f916Y");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("WiFi connected");
}

void loop() {
  buttonState = digitalRead(buttonPin);
  Serial.print("Estado del botón: ");
  Serial.println(buttonState);
  
  // Encender LED si el botón está presionado
  if (buttonState == HIGH) {
    digitalWrite(ledPin, HIGH);
  } else {
    digitalWrite(ledPin, LOW);
  }
  
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    String message = url + "api_key=" + api_key + "&" + sensor + "=" + String(buttonState);
    
    http.begin(message);
    int httpCode = http.GET();
    
    Serial.println(message);
    Serial.print("HTTP Response Code: ");
    Serial.println(httpCode);
    
    http.end();
  }
  
  delay(5000); // Esperar antes del siguiente envío
}

5. Sensor Analógico (Potenciómetro)

Simulación en WOKWI:

Código Sensor Analógico (Potenciómetro) 
#include <WiFi.h>
#include <HTTPClient.h>

int potPin = 35;
int ledPin = 18;
int potValue = 0;

String url = "https://api.thingspeak.com/update?";
String api_key = "FUV135ZIDNU0UE1K";
String sensor = "field3";

void setup() {
  pinMode(ledPin, OUTPUT);
  Serial.begin(115200);
  
  WiFi.begin("ALFREDOMACHINE", "35|f916Y");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("WiFi connected");
}

void loop() {
  potValue = analogRead(potPin);
  // Normalizar el valor para ESP32 (0-4095 a 0-1024)
  potValue = map(potValue, 0, 4095, 0, 1024);
  
  Serial.print("Valor del potenciómetro: ");
  Serial.println(potValue);
  
  // Modular el brillo del LED según el valor del potenciómetro
  int brightness = map(potValue, 0, 1024, 0, 255);
  analogWrite(ledPin, brightness);
  
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    String message = url + "api_key=" + api_key + "&" + sensor + "=" + String(potValue);
    
    http.begin(message);
    int httpCode = http.GET();
    
    Serial.println(message);
    Serial.print("HTTP Response Code: ");
    Serial.println(httpCode);
    
    http.end();
  }
  
  delay(5000); // Esperar antes del siguiente envío
}

6. Sensor Inteligente (DHT22)

Simulación en WOKWI:

Código Sensor Inteligente (DHT22) 
#include <WiFi.h>
#include <HTTPClient.h>
#include <DHT.h>
#define DHTPIN 32
#define DHTTYPE DHT22
DHT dht(DHTPIN, DHTTYPE);
int led = 33;
int led2 = 25;
int temp_value = 0;
int hum_value = 0;
String url = "https://api.thingspeak.com/update?";
String api_key = "FUV135ZIDNU0UE1K";
String sensor = "field4";
void setup() {
  pinMode(led, OUTPUT);
  pinMode(led2, OUTPUT);
  Serial.begin(115200);
  dht.begin();
  
  WiFi.begin("ALFREDOMACHINE", "35|f916Y");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("WiFi connected");
}
void loop() {
  delay(1000);
  
  temp_value = dht.readTemperature();
  hum_value = dht.readHumidity();
  Serial.print("Temperatura: ");
  Serial.println(temp_value);
  Serial.print("Humedad: ");
  Serial.println(hum_value);
  if (hum_value >= 50) {
    digitalWrite(led, HIGH);
  } else {
    digitalWrite(led, LOW);
  }
  if (temp_value >= 23) {
    digitalWrite(led2, HIGH);
  } else {
    digitalWrite(led2, LOW);
  }
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    String message = url + "api_key=" + api_key + "&" + sensor + "=" + String(temp_value);
    
    http.begin(message);
    int httpCode = http.GET();
    
    Serial.println(message);
    Serial.print("HTTP Response Code: ");
    Serial.println(httpCode);
    
    http.end();
  }
  delay(3000); // Espera antes del siguiente envío
}

7. Código Integrado para Todos los Sensores

Código Integrado para Todos los Sensores 
#include <WiFi.h>
#include <HTTPClient.h>
#include <DHT.h>

// Definición de pines
#define DHTPIN 32
#define DHTTYPE DHT22

const int buttonPin = 26;
const int pirPin = 34;
const int potPin = 35;
const int ledButton = 27;
const int ledPir = 5;
const int ledPot = 18;
const int ledTemp = 33;
const int ledHum = 25;

// Variables para lecturas de sensores
int buttonState = 0;
int pirState = 0;
int potValue = 0;
int temp_value = 0;
int hum_value = 0;

// Configuración de ThingSpeak
String url = "https://api.thingspeak.com/update?";
String api_key = "FUV135ZIDNU0UE1K";

// Inicialización de objetos
DHT dht(DHTPIN, DHTTYPE);

void setup() {
  // Configuración de pines
  pinMode(buttonPin, INPUT);
  pinMode(pirPin, INPUT);
  pinMode(ledButton, OUTPUT);
  pinMode(ledPir, OUTPUT);
  pinMode(ledPot, OUTPUT);
  pinMode(ledTemp, OUTPUT);
  pinMode(ledHum, OUTPUT);
  
  // Inicialización del DHT y Serial
  Serial.begin(115200);
  dht.begin();
  
  // Conexión WiFi
  WiFi.begin("ALFREDOMACHINE", "35|f916Y");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("WiFi connected");
}

void loop() {
  // Leer todos los sensores
  buttonState = digitalRead(buttonPin);
  pirState = digitalRead(pirPin);
  potValue = analogRead(potPin);
  potValue = map(potValue, 0, 4095, 0, 1024); // Normalizar para ESP32
  temp_value = dht.readTemperature();
  hum_value = dht.readHumidity();
  
  // Actualizar LEDs según las lecturas
  digitalWrite(ledButton, buttonState);
  digitalWrite(ledPir, pirState);
  analogWrite(ledPot, map(potValue, 0, 1024, 0, 255));
  
  if (temp_value >= 23) {
    digitalWrite(ledTemp, HIGH);
  } else {
    digitalWrite(ledTemp, LOW);
  }
  
  if (hum_value >= 50) {
    digitalWrite(ledHum, HIGH);
  } else {
    digitalWrite(ledHum, LOW);
  }
  
  // Mostrar lecturas en el monitor serial
  Serial.println("------ Lecturas de Sensores ------");
  Serial.print("Botón: ");
  Serial.println(buttonState);
  Serial.print("PIR: ");
  Serial.println(pirState);
  Serial.print("Potenciómetro: ");
  Serial.println(potValue);
  Serial.print("Temperatura: ");
  Serial.println(temp_value);
  Serial.print("Humedad: ");
  Serial.println(hum_value);
  Serial.println("--------------------------------");
  
  // Enviar datos a ThingSpeak si WiFi está conectado
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    
    // Construir URL con todos los campos
    String message = url + "api_key=" + api_key + 
                    "&field1=" + String(buttonState) + 
                    "&field2=" + String(pirState) + 
                    "&field3=" + String(potValue) + 
                    "&field4=" + String(temp_value) + 
                    "&field5=" + String(hum_value);
    
    http.begin(message);
    int httpCode = http.GET();
    
    Serial.println(message);
    Serial.print("HTTP Response Code: ");
    Serial.println(httpCode);
    
    http.end();
  }
  
  delay(15000); // Esperar 15 segundos antes del siguiente envío
}

- Resultados -

Los resultados se pueden visualizar en los gráficos generados por ThingSpeak:

Sensor 1: Digital (Botón)

Como se observa en la imagen, el sensor digital muestra valores de 0 cuando el botón no está presionado y 1 cuando está presionado. Se puede ver una activación del botón entre las 12:08 y 12:09, manteniendo el valor en 1 durante ese período.

Sensor 2: Digital PIR

El sensor PIR muestra una detección de movimiento constante (valor 1) hasta las 20:41:45, momento en el que deja de detectar movimiento (valor 0), para luego volver a detectar a las 20:42:15.

Sensor 3: Analógico (Potenciómetro)

La imagen muestra las lecturas del potenciómetro, con valores que van desde 0 hasta aproximadamente 4000 (escala del ADC). Se observa un incremento repentino a las 12:12:15, manteniendo el valor máximo durante unos 45 segundos antes de volver a 0.

Sensor 4: Temperatura (DHT22)

Se muestran las lecturas de temperatura, comenzando en 26°C y aumentando gradualmente hasta alcanzar los 28°C al final del período de registro, probablemente debido a factores ambientales o al calentamiento del propio sensor.

Estos gráficos demuestran la correcta implementación de la comunicación entre los sensores, el módulo ESP32/ESP8266 y la plataforma ThingSpeak, cumpliendo así con los objetivos de la práctica.
